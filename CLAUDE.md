# 中国象棋项目 AI 工作指引

## 项目定位
轻量化本地中国象棋应用，纯前端单 HTML 文件，零依赖，双人本地对弈。

## 重要文档索引

| 文档 | 路径 | 用途 |
|------|------|------|
| 开发需求 | `docs/requirements.md` | 功能需求和非功能需求清单 |
| 技术架构 | `docs/architecture.md` | 数据模型、模块划分、渲染流程 |
| UI 设计 | `docs/ui-design.md` | 色彩方案、布局、交互规范 |
| 开发规范 | `docs/standards.md` | 编码规范、开发流程 |
| 执行步骤 | `docs/dev-steps.md` | 分阶段开发计划，当前进度 |
| 开发日志 | `devlog/YYYY-MM-DD.md` | 每日完成/待办记录 |

## 工作流程
1. **每次开始工作前**：先阅读 `docs/dev-steps.md` 确认当前阶段
2. **编码前**：阅读 `docs/architecture.md` 和 `docs/standards.md` 确保符合架构和规范
3. **编码后**：验证功能正常（无控制台错误，交互符合预期）
4. **每次会话结束时**：更新 `devlog/` 中当日的日志文件

## 开发原则
- **逐阶段推进**：严格按 `docs/dev-steps.md` 的阶段顺序推进
- **每步可验证**：每个步骤完成后确保功能可运行、可验证
- **先核心后外围**：先核心游戏逻辑，再 UI 辅助功能
- **单文件部署**：所有代码在 `index.html` 一个文件内
- **零外部依赖**：不使用任何外部库或 CDN 资源

## 语言规范
- **重要：** 你的所有思考过程使用简体中文。
